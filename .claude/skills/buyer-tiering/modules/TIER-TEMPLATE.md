# Tier Definition Template

Output format for buyer tiering.

## Template

```
# Tier Definitions: {Client Name}

**Product:** {What the client sells}
**Date:** {date}
**Based on:** product-profile.md

---

## Tier 1: In-Market NOW

**Expected response rate:** 15-25%

### Criteria
- {Observable signal 1}
- {Observable signal 2}

### Where to Find
- {Data source 1}: {What to search for}
- {Data source 2}: {What to search for}

### Volume Estimate
{Estimated number of prospects matching these criteria}

### Messaging Approach
{How to address these prospects - specific, direct, value-first}

---

## Tier 2: Approaching

**Expected response rate:** 5-10%

### Criteria
- {Leading indicator 1}
- {Leading indicator 2}

### Where to Find
- {Data source 1}: {What to search for}
- {Data source 2}: {What to search for}

### Volume Estimate
{Estimated number of prospects}

### Messaging Approach
{How to address these prospects - educational, insight-driven}

---

## Tier 3: ICP Fit

**Expected response rate:** 1-3%

### Criteria
- {Firmographic criteria 1}
- {Firmographic criteria 2}

### Where to Find
- {Standard database filters}

### Volume Estimate
{Estimated number of prospects}

### Messaging Approach
{How to address these prospects - proof points, general value prop}
```

## Quality Checks

- Tier 1 must have at least ONE observable, findable signal
- Tiers must be mutually exclusive (a prospect fits one tier, not multiple)
- Volume estimates should be realistic (Tier 1 is always the smallest)
- Messaging approach must be distinct per tier (not the same email with different data)
