# Product Profile Template

Output format for prospect research.

## Template

```
# Product Profile: {Company Name}

**Website:** {url}
**Research date:** {date}

## Problem
{What pain does this product solve? Be specific.}

## Solution
{How does the product solve it? Key capabilities.}

## Value Propositions
1. {Primary value prop with specific claim}
2. {Secondary value prop}
3. {Tertiary value prop}

## Ideal Buyer

| Attribute | Details |
|-----------|---------|
| Title | {Decision maker roles} |
| Company type | {Industry, size, stage} |
| Situation | {When they buy - what's happening in their business} |

## Buying Triggers

| Trigger | Signal | Where to Find |
|---------|--------|---------------|
| {Trigger 1} | {Observable data point} | {Source: job boards, LinkedIn, news} |
| {Trigger 2} | {Observable data point} | {Source} |
| {Trigger 3} | {Observable data point} | {Source} |

## Existential Data Point (EDP)

**The ONE signal that proves they need this NOW:**

{Description of the EDP}

**Where to find it:** {Data source}
**Why it matters:** {What this signal tells us about their situation}

## Proof Points

- {Case study, testimonial, or data point from their website}
- {Another proof point}

## Notes

{Any additional observations, competitive landscape, market context}
```

## Quality Checks

- Every trigger must be OBSERVABLE (you can find it in public data)
- No vague triggers like "companies that struggle with X" - be specific
- EDP must be a single, findable data point
- Value props must include specific claims (numbers, comparisons, outcomes)
